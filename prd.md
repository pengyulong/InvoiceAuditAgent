# 智能合同发票审计系统 PRD v2.0 (MVP精简版)

## 1. 产品概述

### 1.1 产品定位
基于开源多模态大模型和LLM的智能合同发票审计系统，通过简化的Agentic Workflow自动化审计采购合同与发票的匹配性、合规性和准确性。专为MVP验证设计，快速验证核心价值 proposition。

### 1.2 核心价值
- **自动化审计**: 替代人工审核，提高审计效率90%以上
- **智能分析**: 利用多模态大模型理解合同和发票图像内容
- **风险识别**: 自动发现重复发票、金额不符等异常问题
- **快速验证**: 单次审计流程控制在60秒内完成

### 1.3 目标用户
- 中小型企业财务审计人员
- 采购管理部门
- 财务合规团队（1-3人团队规模）

## 2. 技术架构（MVP简化版）

### 2.1 整体架构
```
前端层 (HTML5 + JavaScript)
    ↓
API网关层 (FastAPI - 模拟API)
    ↓
Agent编排层 (简化版LangGraph逻辑)
    ↓
模型服务层 (阿里云多模态API + DeepSeek API)
    ↓
临时存储层 (浏览器缓存 + JSON文件)
```

### 2.2 核心技术栈

#### 前端
- **框架**: 原生HTML5 + CSS3 + JavaScript (ES6+)
- **UI库**: 无依赖，原生实现
- **动画**: Anime.js, Matter.js
- **图表**: ECharts.js
- **样式**: Tailwind CSS (CDN)

#### 后端（模拟）
- **语言**: JavaScript (前端模拟)
- **API**: 模拟FastAPI响应
- **数据处理**: 前端JavaScript处理
- **文件处理**: 浏览器File API

#### AI模型（实际调用）
- **OCR模型**: 百度云PaddleOCR API
- **推理LLM**: DeepSeek API
- **备用方案**: 本地mock数据（演示模式）

#### 存储（简化）
- **会话存储**: localStorage/sessionStorage
- **文件缓存**: 浏览器临时存储
- **结果存储**: JSON格式内存缓存

### 2.3 Agent架构设计（MVP版）

#### 简化Agent协作
```
用户上传 → 协调器 → 并行处理 → 结果汇总 → 报告生成
    ↓           ↓        ↓         ↓         ↓
  文件解析   合同分析   发票分析   交叉验证   结果展示
```

#### Agent职责定义

**1. 协调器Agent (Coordinator)**
- 文件上传和初步验证
- 任务分发和状态管理
- 进度跟踪和错误处理
- 结果汇总和报告生成

**2. 合同分析Agent (Contract Analyzer)**
- 提取合同关键信息（合同编号、金额、税率等）
- 合同合理性评估
- 商品清单解析
- 输出结构化数据

**3. 发票分析Agent (Invoice Analyzer)**
- 批量发票信息提取
- 重复发票检测
- 发票错误识别
- OCR置信度评估

**4. 交叉验证Agent (Cross-Validator)**
- 合同-发票信息匹配
- 金额一致性验证
- 覆盖率计算
- 差异项识别

**5. 审核报告Agent(reporter)**
- 总结合同关键信息
- 统计发票关键信息
- 分析结论和风险建议

## 3. 功能需求（MVP核心功能）

### 3.1 文件上传管理
- **F1.1** 支持上传ZIP文件（1份合同 + 多份发票），大小≤50MB
- **F1.2** 文件类型自动识别和验证
- **F1.3** 上传进度实时显示
- **F1.4** 文件预览和基本信息展示

### 3.2 合同分析
- **F2.1** 自动提取合同关键字段
  - 合同编号、买卖方信息
  - 合同总金额、税率
  - 商品清单（品名、规格、数量、单价）
- **F2.2** 合同合理性评估
  - 金额计算准确性验证
  - 基础价格异常检测
  - 合同要素完整性检查

### 3.3 发票分析
- **F3.1** 批量发票信息提取
  - 发票代码、号码、日期
  - 买卖方信息、金额、税率
- **F3.2** 发票重复检测
  - 基于发票号码的精确匹配
  - 基于内容的模糊匹配
- **F3.3** 发票错误检测
  - 金额计算错误检查
  - 必填字段完整性验证

### 3.4 合同-发票交叉验证
- **F4.1** 合同编号匹配验证
- **F4.2** 买卖双方信息匹配
- **F4.3** 商品清单匹配验证
- **F4.4** 金额一致性验证
- **F4.5** 发票覆盖率计算

### 3.5 审计报告生成
- **F5.1** 综合审计结论（通过/不通过/需人工复核）
- **F5.2** 问题清单分类（严重/警告/信息）
- **F5.3** 详细对比表格展示
- **F5.4** 报告导出功能（PDF/Excel/JSON）

### 3.6 实时交互
- **F6.1** 审计进度实时显示
- **F6.2** Agent工作过程可视化
- **F6.3** 基础查询和解释功能
- **F6.4** 审计历史记录（会话级）

## 4. 非功能需求

### 4.1 性能要求
- **响应时间**: 单次完整审计<60秒

- **系统可用性**: >95%（排除API故障）

### 4.3 用户体验要求
- **浏览器兼容**: Chrome 90+, Firefox 88+, Safari 14+
- **响应式设计**: 支持桌面端（1920x1080）和平板端
- **加载时间**: 首屏<3秒

### 4.4 安全要求

- **数据保护**: 本地处理，不持久化存储
- **访问控制**: 无需注册，直接使用
- **审计日志**: 本地操作记录
